1. **Challenge: Understanding Django's MVT Architecture**

2. **Challenge: Django ORM vs MongoDB**
   - Relational database concepts after using NoSQL
   - Solution: Learned about database migrations, foreign keys, and one-to-many/many-to-many relationships

3. **Challenge: Cart Consolidation Bug**
   - Multiple cart instances created for same user causing integrity errors
   - Solution: Implemented smart cart merging logic and created management commands for data cleanup

4. **Challenge: Stock Management**

   - Solution: Built real-time validation, stock snapshots, and automatic quantity adjustment

5. **Challenge: Template System**

**Django vs MERN - Key Differences I Discovered:**

**1. Architecture:**
- MERN: Separate frontend (React) and backend (Node/Express) - requires API calls
- Django: Monolithic - templates rendered server-side, less API overhead

**2. Database Handling:**
- MERN: MongoDB (NoSQL) - flexible schemas, manual relationship handling
- Django: SQL with ORM - built-in migrations, automatic relationship management, stronger data integrity

**3. Authentication:**
- MERN: Manual implementation (JWT, passport.js, custom middleware)
- Django: Built-in auth system out-of-the-box with user models, sessions, and security

**4. Admin Panel:**
- MERN: Build from scratch or use third-party libraries
- Django: Automatic admin interface - massive time saver!

**5. Learning Curve:**
- MERN: More JavaScript knowledge required, separate frontend/backend learning
- Django: "Batteries included" - everything in one framework, faster for MVPs

**6. Development Speed:**
- MERN: More setup initially, but flexible for SPAs
- Django: Rapid development for traditional web apps, faster to production

**7. Templating:**
- MERN: JSX/React components - highly dynamic
- Django: Template language - simpler for server-rendered pages

**8. Real-time Features:**
- MERN: Easier with Socket.io and Node's event-driven nature
- Django: Requires Django Channels for WebSockets

**9. API Development:**
- MERN: Express.js - minimal, more manual work
- Django: Django REST Framework - serializers, viewsets, built-in browsable API

**10. Deployment:**
- MERN: Often separate (Vercel/Netlify for frontend, Heroku/AWS for backend)
- Django: Single deployment, but requires proper WSGI/ASGI server setup
